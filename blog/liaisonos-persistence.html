Persistence — Your Settings Follow Your Key

How LiaisonOS saves your callsign, radio configs, Winlink mailbox, maps and more on your USB key — and restores everything automatically at every boot.

Sylvain Deguire (VA2OPS) · LiaisonOS 2.2.7

The idea

LiaisonOS can run entirely from a USB key — nothing installed on the computer you plug it into. Boot any machine, be operational in minutes, leave no trace behind — and no alteration on the host PC. For best performance, a permanent install on an internal drive or SSD is recommended — but the USB live mode is what makes LiaisonOS unique in the field. Either way, the question is the same: where do your settings go when you shut down?

The answer is the DATA partition on your USB key. Every time you shut down, LiaisonOS saves your configurations, your Winlink mailbox, and your personal data directly to that partition. The next time you boot — on the same machine or a completely different one — everything is restored automatically before the desktop even appears.

Your key is your station. It goes with you.

Maps and Wikipedia are different. Large files like offline maps and Kiwix ZIM files are not copied into memory — they are accessed directly from the DATA partition via symlinks. This keeps RAM usage low regardless of how much data you carry on the key.

How it works at boot

When LiaisonOS starts, it looks for a DATA partition on the USB key before the desktop loads. Two things happen automatically:

Boot LiaisonOS │ ├─► Maps & Wikipedia symlinked from DATA → available instantly, no RAM used │ └─► et-firstboot checks DATA partition │ ├─ Config found? ──► Restore all settings ──► "Welcome back, VA2OPS" │ └─ No config? ───► First-boot wizard ──────► Set callsign, grid, select DATA partition

On first boot, the wizard asks for your callsign, grid square, and which partition to use as DATA. After that, every subsequent boot is silent and automatic — your station is ready within seconds of the desktop appearing.

How to save

Saving is built into your shutdown workflow. When you are done for the session, use any of the three options from the LiaisonOS menu:

  • Save and Shut Down — saves everything, then powers off
  • Save and Reboot — saves everything, then reboots
  • Save and Suspend — saves everything, then suspends

The save runs during the shutdown sequence — you see the progress on screen. If you forget to save and just power off, your last saved state is still on the key — you just lose any changes made since the previous save.

Do not pull the key while saving. The save writes directly to the DATA partition. Removing the key mid-save can corrupt your configs. Always wait for the shutdown to complete.

On the DATA partition

From your computer, the DATA partition looks like a regular USB drive. Inside, LiaisonOS maintains a clean directory structure:

DATA/ └── et-config/ ├── user.json ← your callsign and grid ├── manifest.json ← metadata and version ├── configs/ ← one folder per application │ ├── pat/ │ ├── js8call/ │ ├── wsjtx/ │ ├── fldigi/ │ └── ... ├── mailbox/ ← Winlink messages └── user-data/ ← Documents, Downloads, Pictures

You can browse this folder from any computer — Windows, Mac, or Linux — since DATA is formatted as exFAT. This also means you can manually copy the contents of your DATA partition to a second location if you want an extra safety net.
